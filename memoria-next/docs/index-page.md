# Index ページ設計メモ

## 位置づけ

`/` (トップページ) は現在 `AuthScreen` を直接返している暫定状態。
LP (`/lp`) とは役割を分ける。

| ページ | 役割 |
|--------|------|
| `/`    | サービスの入口。未ログインでも見られる。メモリアたんのデモ＋概要＋動線 |
| `/lp`  | 世界観・思想・詳細説明。じっくり読む人向け |

---

## コンセプト

### メモリアたんのプロフィールをデモとして見せる

- 仮想キャラクター「メモリアたん」の実際のプロフカードを表示
- ユーザーは「こういうカードが作れる」を説明文ではなく **実物** で理解する
- メモリアたんはオーナーが実アカウントとして作成（QRも実際にスキャンできる）
- イラスト素材: `public/images/` 内のメモリアたんイラスト（縦長・黒背景・アニメ調）
- アバター画像はイラストから顔部分をトリミングして使用予定

### eyecatch1 をここで使う
- `public/images/eyecatch1.png`（イラスト合成・かわいい方向）
- LP ではクリーン方向に振ったので、index のかわいい世界観担当として活用

## ページ構成案

```
[ヒーロー／メモリアたん紹介]
  - メモリアたんのプロフカード（実物デモ）
  - eyecatch1 or メモリアたんイラストをビジュアルに
  - "会った人を覚えておくプロフィール帳" キャッチ
  - [プロフ帳を作る] [ゲストで試す] CTA

[サービス更新情報]
  - 新機能・アップデートの一覧
  - データソース: コード内配列（MDファイル or ハードコード）
  - CMSは使わない方針

[Memoriaって何？]
  - /lp の抜粋（3〜4行 + "詳しく見る → /lp" リンク）

[よくある質問 FAQ]
  - Q&A 5〜7件程度
  - 将来的には /faq ページへの動線も

[CTA]
  - プロフ帳を作る / ログイン
```

## 更新情報のデータ形式

CMSなし・コード管理で運用する。形式例：

```typescript
// src/data/updates.ts
export const UPDATES = [
  {
    date: "2026-05-14",
    label: "new",           // "new" | "fix" | "info"
    text: "シール編集機能をリニューアル",
  },
  {
    date: "2026-05-01",
    label: "fix",
    text: "フレーム画像がテキストの上に被る問題を修正",
  },
];
```

## 実装上の注意

- `/` はログイン済みなら `/mine` にリダイレクトするか、indexをそのまま見せるか要検討
  - 案A: ログイン済み → `/mine` へ自動リダイレクト
  - 案B: ログイン済みでも index を見せる（topbarにマイページリンク）
  - **暫定方針: 案A** （AuthScreen の redirectOnAuth 動作をそのまま踏襲）

- メモリアたんのアカウント slug は `/profile/memoriatan` などに固定
- イラストの黒背景問題: アバター画像として使う場合はトリミング or カードテーマをダークに
